Media Word Filter — Jellyfin Web

Mutes filtered phrases during Jellyfin Web playback using the MWF mute API (GET /mutes/{item_id}?user_id=…). Fetches run in the extension background worker (no CORS dependency on Jellyfin).

Must be reachable from this browser (use LAN IP, not 127.0.0.1, when MWF runs on another machine). Set the same public base URL in MWF Settings if you use personalized profiles.
Leave blank to use ApiClient.getCurrentUserId() / stored Jellyfin credentials. Override for testing or multi-account edge cases.
Manual debug override; skips URL/playback discovery when set.
Default 100 ms (same order as JMP Lua poll_interval=0.1).

Host permissions cover http://*/* and https://*/* so the background worker can reach your MWF server and content scripts can run on your Jellyfin URL (including custom ports and LAN IPs).
